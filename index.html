Docker 101

Un container è un’istanza runtime di un’immagine Docker. In genere, i requisiti per il disegno di un container sono semplici: deve essere portabile e con un footprint molto contenuto, orientato allo sviluppo e al deployment di applicazioni che si propongono di avere uno stile architetturale moderno. Docker, come dispositivo di automazione, utilizza l'isolamento delle risorse del kernel Linux (cgroups, namespaces) per la coesistenza di più container sulla stessa istanza Linux.

Kubernetes 101

Kubernetes è responsabile della gestione di cluster di container, su Bare Metal o su VM: una piattaforma per il deployment automatico, la schedulazione, lo scaling e le operations di gruppi di container. Open Source, e disponibile su una pluralità di ambienti Cloud, dal 2017 diventa il riferimento primario per gli orchestratori. Oracle utilizza Kubernetes up-stream in Oracle Cloud Infrastructure per il servizio OKE (Oracle Kubernetes Engine).

FnProject 101

Le architetture Serverless sono un'astrazione dell'infrastruttura e delle sue operazioni (provisioning, patching, scaling, etc.), e si riferiscono ad un set applicativo costruito interamente su componenti serverless come compute, storage e networking. Il FaaS (Function as a Service) è il componente compute in un'architettura serverless. FnProject è un progetto Open Source, semplice da apprendere ed elastico (dal Laptop al Cloud). La versione industrializzata (Oracle Functions) è disponibile su Oracle Cloud Infrastructure.
